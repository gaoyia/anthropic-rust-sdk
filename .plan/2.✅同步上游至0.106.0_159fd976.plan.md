# 同步上游 anthropic-sdk-typescript 至 0.106.0

## 背景与关键判断

- 上游当前最新 release 即 `sdk-v0.106.0`（2026-06-24），其后无更新版本。子模块现固定在 `sdk-v0.104.1`。
- Rust 侧大量采用开放结构，导致多数「类型层」变更**自动兼容、无需改代码**：
  - `model: String`（[src/resources/messages/types.rs](src/resources/messages/types.rs) 第 142 行）→ 退役模型移除、新模型均无影响。
  - `tools: Option<Vec<Value>>`（同文件第 160 行）→ 新增 `code_execution_20260521` 工具、`allowed_callers` 新值、工具新字段（`cache_control`/`defer_loading`/`strict`）均无影响。
  - `ContentBlock { type, fields: Value }`（第 80 行）→ refusal 新增 `military_weapons` 类别无影响。
  - `RawMessageStreamEvent { type, fields: Value }`（第 219 行）→ 新事件类型可被容纳。
  - `MessageCreateParams` 含 `#[serde(flatten)] extra: Value`（第 167 行）→ 任意新增 body 字段可透传。
- 真正需要落地的是少数「行为层」改动，集中在阶段 A。

## 上游 0.104.2 - 0.106.0 变更归类

- 自动兼容（不改）：退役模型移除、`code_execution_20260521` 工具与字段、refusal `military_weapons`、大量 "Body param:" 注释、TS 内部文件迁移。
- 需改动（阶段 A，必须）：
  - `user_profile_id` 从请求体抽出为 `anthropic-user-profile-id` 请求头（见上游 [messages.ts diff](anthropic-sdk-typescript/src/resources/messages/messages.ts)：`const { user_profile_id, ...body } = params;`）。
  - `system.message` 流式事件需放行（上游 [src/core/streaming.ts](anthropic-sdk-typescript/src/core/streaming.ts) 新增 `sse.event === 'system.message'`）。
  - 子模块指针与版本/文档同步。
- 运行时增强（阶段 B/C，后续逐轮）：`x-stainless-helper` append 头机制、流式惰性 tool JSON 累积、client 端 fallback middleware。

## 阶段 A：对齐到 0.106.0（本次重点，多个小提交）

### A1 更新子模块指针
- 在子模块目录执行 `git checkout sdk-v0.106.0`（标签对象已拉取），父仓库 `git add anthropic-sdk-typescript` 提交指针。[.gitmodules](.gitmodules) 文件本身无需改（URL 不变、无 branch 配置）。
- 提交（建议）：`build(submodule): 同步子模块至 sdk-v0.106.0 / Bump submodule to sdk-v0.106.0`

### A2 user_profile_id 请求头
- [src/resources/messages/types.rs](src/resources/messages/types.rs)：为 `MessageCreateParams` 与 `MessageCountTokensParams` 新增 `user_profile_id: Option<String>`，标注 `#[serde(skip_serializing)]`（不进 body），并在 `new()` 初始化。
- [src/client.rs](src/client.rs)：新增可携带额外请求头的方法（如 `post_with_headers` / `post_streaming_with_headers`），或泛化现有 `request` 链路；beta 侧扩展 `post_beta` 以并入额外头（现有 [request_beta](src/client.rs) 第 259 行已具备 extra_headers 合并能力，可复用其模式）。
- [src/resources/messages/mod.rs](src/resources/messages/mod.rs)：`create` / `count_tokens` 提取 `user_profile_id` 注入头。
- [src/runtime/message_stream.rs](src/runtime/message_stream.rs)：`events()` 的流式请求同样注入头。
- [src/resources/beta/messages.rs](src/resources/beta/messages.rs)：`create` / `count_tokens` 注入头（注意现有 `create` 已手工序列化 body，需从中剔除并改走带头方法）。
- 测试：用 `wiremock` 断言请求含 `anthropic-user-profile-id` 且 body 不含该字段。
- 提交（建议）：`feat(messages): 支持 user_profile_id 请求头 / Add user_profile_id request header`

### A3 放行 system.message 流式事件
- [src/internal/sse.rs](src/internal/sse.rs)：在 `is_message_stream_event`（第 83-95 行）白名单加入 `"system.message"`，使其不被 `EventStream` 过滤。
- 测试：构造含 `system.message` 的 SSE fixture，断言事件能透传为 `RawMessageStreamEvent`。
- 提交（建议）：`feat(streaming): 透传 system.message 流式事件 / Pass through system.message stream events`

### A4 版本与文档同步
- [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) 第 78 行：参考版本 `0.104.1` 改为 `0.106.0`。
- [docs/ROADMAP.md](docs/ROADMAP.md)：新增「上游同步 0.104.2–0.106.0」小节，记录已对齐项，并将运行时增强标注为延期（💤）。
- [README.md](README.md)：如含版本说明则同步。
- 可选：[Cargo.toml](Cargo.toml) version 适度 bump（如 0.1.1）。
- 提交（建议）：`docs: 同步上游版本至 0.106.0 并更新路线图 / Sync upstream version to 0.106.0`

### A5 打 tag（对齐完成后）
- 给本仓库打与上游对应的 tag，建议命名 `sdk-v0.106.0`（与上游一致）或 `sync/sdk-v0.106.0`；实施时确认命名。
